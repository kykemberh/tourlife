# TourLife

Соціальна мережа для подорожей: реєстрація, стрічка постів, лайки, коментарі, особисті повідомлення.

Без зовнішніх npm-залежностей — лише вбудовані модулі Node.js (потрібен Node.js версії 22.5 або новішої, через вбудований `node:sqlite`).

## Запуск локально

```
node server.js
```

Відкрити http://localhost:3000

За потреби порт можна змінити: `PORT=4000 node server.js`

## Деплой в інтернет (безкоштовно, Render.com)

1. Створити акаунт на https://render.com (можна через GitHub).
2. Завантажити цю папку в новий репозиторій на GitHub (через сайт GitHub: "Create repository" → "uploading an existing file", або через git, якщо встановлено).
3. У Render: "New +" → "Web Service" → підключити репозиторій.
4. Налаштування:
   - Build Command: (залишити порожнім)
   - Start Command: `node server.js`
   - Environment: додати змінну `TOURLIFE_SECRET` з будь-яким довгим випадковим значенням (це секрет для підпису токенів входу).
   - Переконатися, що обрано Node.js версії 22 (Render Environment → Node Version, або додати файл `.node-version` з вмістом `22.13.0`).
5. Натиснути "Create Web Service" — Render видасть публічну адресу типу `https://tourlife.onrender.com`.

Альтернативи з аналогічним безкоштовним рівнем: Railway.app, Fly.io.

## Важливо знати

- Дані (користувачі, пости, повідомлення) зберігаються у файлі `data/tourlife.sqlite` на сервері. На безкоштовному тарифі Render диск може очищатися при перерозгортанні — для постійного зберігання даних варто підключити Render Persistent Disk (платна опція) або перейти на хмарну БД пізніше.
- Фото/відео в композері — наразі лише позначка з назвою файлу (демо), реального завантаження на сервер ще немає.
- Це MVP. Перед реальним публічним запуском варто додати: підтвердження email, обмеження частоти запитів (rate limiting), політику конфіденційності та умови використання.
